# Workflow: Write 

> **写作哲学**：大纲是指南针，不是施工图。正文写作中允许意外之喜——如果把灵感先写进去，再评估是否调整后续大纲。

## 步骤 0: 一次性初始化 (One-time Setup)

*   **目标**：以主角的初始状态作为整个创作阶段的基准。（拆件初始化时已创建相关文件，本步仅做更新与链接。）
*   **动作**：首次创作前，根据 `XX卷_章纲.md` 的开局设定：
    1.  **更新初始状态**：在 `.novelkit/memory/character_state.md` 中记录/更新主角第 0 章（正文开始前）的初始状态。
    2.  **链接到 SOLOENT**：确认 `SOLOENT.md §7.2 Character Snapshot` 已写入指针 `-> .novelkit/memory/character_state.md`；若未写入则补上。


---

### 步骤 1：写作前置检查 (Pre-Writing Check)
*   **目标**：确保动笔前的核心信息输入到位。
*   **动作**：在撰写 **每一章** 正文前，必须检查并整合以下关键信息：
    1.  **风格约束**：
        *   `1-边界/1.2_文风.md` (基础文风 — 路径已注册于 `SOLOENT.md §5`)
        *   `1-边界/1.5_微观节奏拆解.md` (如有：导演手册，场景调度/镜头语言)
    2.  **红线底线**：`docs/MASTER.md` (避坑指南 — 路径已注册于 `SOLOENT.md §6`)
    3.  **设定支持**：
        *   查阅 `SOLOENT.md §2`（世界观索引）和 `§3`（角色索引），按指针读取对应的世界观和角色档案文件
        *   `2-设定/2.3_金手指设定.md` (如有：道具说明书，若本章涉及使用)
    4.  **当前状态**：`.novelkit/memory/character_state.md`（路径注册于 `SOLOENT.md §7.2`，确保获取最新状态）
        *   **同时读取** `🔍 情报错位追踪` 节（若存在信息差，在写正文前确认角色间的信息差）
    5.  **伏笔与进度**：
        *   `.novelkit/memory/foreshadowing.md`（路径注册于 `SOLOENT.md §7.4`）
        *   **同时读取** `🆕 主角待办清单` 节（若存在短期待办项，写作时确保主角有所行动）
        *   `SOLOENT.md §8`（当前阶段与待办）
    6.  **本章指令**：`3-大纲/XX卷_章纲.md` 中对应的本章章纲部分。
    7.  **上下文参考**：读取 **上一章正文** (确保剧情连贯、语气承接)。
    8.  **创作偏好**：读取 `1-边界/章纲精修偏好清单.md` §2 和 §3，获取叙事手段比例、对话占比等参数，辅助写作中的节奏平衡。
    9.  **弧段压力参考**：`SOLOENT.md §8.2`（若已切分弧段，确保本章情绪节奏与弧段压力曲线对齐）
    
*   **目的**：防止逻辑BUG、风格漂移和设定遗忘。

### 步骤 2：正文撰写与AI辅助 (Writing with AI)
*   **输入**：步骤1中整合的全部信息。
*   **动作**：
    *   **分场景创作，锚点驱动**：严格按照章纲中定义的**场景结构**顺序，**逐个场景**进行撰写。每个场景内部围绕该章节的**情绪目标**和**核心剧情点**自由展开。不跳过场景，不随意调整场景顺序。
    *   **字数锚定**：建议单章总字数在 **2300-4000 字** 之间（用户可根据平台要求自行指定范围）。
    *   **叙事手段平衡**：参考步骤1中读取的叙事手段比例参数，在对话/描写/战斗/文戏之间合理分配篇幅。若用户未设置比例，以对话为主（建议 ≥ 40%）。
    *   **文风执行**：严格执行 `1.2_文风.md` 的要求，注重"展示而非讲述"(Show, Don't Tell)，强化感官描写。
    *   **情报错位利用**：若步骤1中存在情报错位条目，在写作中体现信息差导致的角色判断/行动差异；写作完成后清除已使用的条目。
    *   **主角待办驱动**：若 `.novelkit/memory/foreshadowing.md` 中有短期待办项，确保本章主角在行动上有所推进（至少提及进展，不要求一定完成）。
    *   **结尾钩子**：确保章节末尾包含章纲中设计的悬念或钩子。
    *   **允许意外捕获**：写作中如果出现新灵感（配角抢戏/伏笔自动闭环/新剧情生长点），**先写进去**，在步骤3中评估是否需要调整后续大纲。**不允许超出的是剧情锚点（章纲定义的核心事件），而不是表现方式。**
*   **输出产物**：`4-正文/第X章_草稿.md`。

### 步骤 3：完稿后校准 (Post-Writing Calibration)
*   **动作**：
    1.  **严格审查（四项核心 + 两项扩展）**：
        *   **字数检查**：确保单章字数在2300-4000字之间（使用脚本 `scripts/count_chinese_words.py` 进行统计），可以超出 4000 字，但是不能低于 2300 字。
        *   **🎯 情绪目标达成检查（替代旧有的"大纲符合性检查"）**：对照本章章纲中的情绪目标，检查正文是否有效触达了该情绪（紧张/爽/期待/压抑后释放/意外反转）。**若未触达**，AI 须给出重新调整结尾或关键段落的建议。
        *   **文风检查**：对照 `1.2_文风.md` 和 `docs/MASTER.md`，检查是否存在AI写作痕迹。
        *   **接续检查**：对照上一章，检查剧情、时间、地点是否无缝接续。
        *   **【扩展】叙事手段检查**：对照步骤1中读取的叙事手段比例参数，检查本章的对话/描写/战斗/文戏占比是否符合预期（若用户未设置此偏好，跳过）。
        *   **【扩展】情报错位检查**：若本章存在信息差，检查"角色知道什么"与"角色判断/行动"之间的因果关系是否一致。
        *   **【扩展】金手指合规性**：(若涉及) 检查表现形式是否符合 `2.3 金手指设定.md`。

    2.  **显式输出【完稿自检卡】**：
        ```markdown
        【完稿自检卡】
        ✅ 字数：[X]字（目标 2300-4000）
        ✅ 情绪目标：[达成/未达成]
        ✅ 文风检查：[通过/未通过]
        ✅ 接续检查：[通过/未通过]
        ✅ [扩展项，如涉及]
        ```
        如果字数检查或情绪目标检查有一项不通过，必须重新创作或调整；文风或接续不通过至少需提示用户。

    3.  **🎯 灵感捕获（可选）**：
        在自检卡之后，增加一个可选分支：
        ```
        ## 🎯 灵感捕获

        写作中是否出现了大纲未预设的"意外之喜"？
        - [ ] 是 → 记录灵感点：[1-2句话描述本次捕获的新灵感]，并询问用户：
          - "是否将本次灵感纳入后续章纲？"
          - "是否需要调用 novel-outline 步骤5（动态修订）进行大纲更新？"
        - [ ] 否 → 直接进入下一步
        ```
*   **目的**：保持设定与正文的实时同步，确保单章质量达标；同时不堵死灵感的自然涌现。

### 步骤 4：伏笔、状态与进度更新 (Foreshadowing, State & Progress Update)
*⚠️ **CRITICAL STEP**: 此步骤决定了记忆的连续性。绝对不可跳过！*

*   **状态回写**：根据本章发生的实际剧情，**立即更新**以下文件：
    *   `.novelkit/memory/character_state.md` → 角色状态变化（升级/道具/关系），并按 `📝 增量更新指引` 执行：
        - [ ] 更新主角状态节（位置/等级/道具/目标/关系，仅变化部分）
        - [ ] 更新副主角/配角状态节（若本章涉及）
        - [ ] 清空"本轮信息差"条目，追加新产生的信息差（若有）
        - [ ] 写入剧情影响摘要（若主角处境有重大变化）
    *   `.novelkit/memory/foreshadowing.md` → 新增或收回的伏笔条目，并按 `🆕 主角待办清单` 更新规则清理/追加待办项
    *   `SOLOENT.md §8.1 & §8.2` → 将本章标记为已完成，更新弧段完成状态（若适用），填入下章任务
*   **执行确认**：必须明确输出"已更新 memory 文件"的反馈，并列出实际更新的字段（如"主角状态：§7.2 已更新位置；待办：已标记'调查黑铁片'为进行中"），方可结束本章创作循环。

## 🚫 循环阻断 (Loop Break)
*   **动作**：在完成上述所有步骤（1-4）后，**必须暂停**。
*   **状态检查与分支选择**：
    1.  **正常推进**：
        - 检查当前卷 `XX卷_章纲.md`。
        - **有剩余章节**：询问 "第 X 章完成。是否继续第 X+1 章？"
        - **本卷已完**：提示 "本卷完。请调用 `novel-outline` 规划下一卷。"
    2.  **剧情偏离/修订**：
        - 若本章剧情与章纲差异较大，或用户表示"有了新想法"。
        - **引导**："检测到/你提到剧情有变化，是否调用 `novel-outline` 的【动态修订】模块，重做后续大纲？"
*   **等待**：等待用户明确指令，严禁自动开始下一章。
