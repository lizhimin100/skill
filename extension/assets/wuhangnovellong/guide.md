# Workflow: Guide
> **目标**：打破线性流程，支持“半途接入”或“带资进组”。将用户已有的灵感、文件、片段整合进标准工作流。
> **适用场景**：
> 1. 用户已有部分稿件（大纲、人设、正文存稿）。
> 2. 用户不想从样板书或纯脑暴开始，而是想从“人设”、“世界观”等特定点切入。

---

## 🧭 模式选择

**触发指令**：`整理项目`, `导入旧稿`, `灵活开始`, `guide`

**AI 动作**：
主动询问用户当前的情况：
> "请问您目前的创作状态是？
> 1. **带资进组（存量整合）**：根目录下已经有一些文件（大纲、人设、草稿），需要整理并继续写。
> 2. **灵感切入（非线性启动）**：没有文件，但想从特定角度（如先写人设、先写黄金三章）开始。
> 3. **常规流程**：返回标准模式或者存量/灵活模式。"

---

## 📂 模式 A: 带资进组 (存量整合)

**目标**：扫描用户目录下的现有文件，将其“搬家”或“注册”到 `.novelkit` 标准体系中，以便后续技能调用。

### 步骤 1: 资产扫描 (Scan)
1.  **执行**：列出项目根目录下的所有文本文件（.md, .txt, .doc, .docx）。
2.  **排除**：忽略 `.novelkit/`, `skills/`, `SOLOENT.md` 等系统文件。
3.  **展示**：向用户展示发现的文件列表。

### 步骤 2: 智能识别与映射 (Identify & Map)
**AI 动作**：
逐个读取文件内容（摘要），判断其属性，并建议归档位置：

| 文件内容特征 | 建议归档位置 | 对应技能模块 |
| :--- | :--- | :--- |
| 样板书、同人文参考 | `1-边界/` | Analyze |
| 故事梗概、核心脑洞 | `2-设定/2.1_创意脑暴.md` | Ideation |
| 世界观、地图、等级体系 | `2-设定/2.2_新书设定案.md` | Ideation |
| 角色介绍、人物小传 | `2-设定/2.4_主要角色设定表.md` | Ideation |
| 金手指、系统说明 | `2-设定/2.3_金手指设定.md` | Ideation |
| 分卷规划、细纲 | `3-大纲/` | Outline |
| 正文稿件 | `4-正文/` | Write |

### 步骤 3: 执行迁移 (Migrate)
1.  **询问**：用户确认归档方案。
2.  **执行**：
    *   **移动/重命名**：将文件移动到对应目录（推荐）。
    *   **或 建立索引**：若用户不愿移动，则在 `SOLOENT.md` 的对应章节（如 `§2 World Index`）中建立指向该文件的**指针**。
3.  **注册**：确保所有识别出的资产都已注册到 `SOLOENT.md` 的索引区。
    *   **世界观** -> 更新 `SOLOENT.md §2`
    *   **角色** -> 更新 `SOLOENT.md §3`
    *   **大纲** -> 更新 `SOLOENT.md §4`

### 步骤 4: 进度定锚 (Anchor)
根据整合后的资产情况，判断当前进度并推荐下一步：
*   *有设定，无大纲* -> 推荐跳转 **Outline**。
*   *有大纲，无正文* -> 推荐跳转 **Write**。
*   *有正文* -> 推荐跳转 **Review** 进行质检，或 **Write** 继续写。

---

## 🎨 模式 B: 灵感切入 (非线性启动)

**目标**：允许用户不按 `Analyze -> Ideation -> Outline` 的顺序，而是直接进入感兴趣的环节，并在完成后引导回主流。

### 场景 1: "我只想先写人设" (Character First)
*   **动作**：
    1.  直接调用 `novel-ideation` 的 **步骤 4 (角色人设细化)**。
    2.  生成 `2-设定/2.4_主要角色设定表.md`。
    3.  **引导回流**："人设已立住。为了让角色有戏，我们需要一个舞台。是否继续完善【世界观】或直接构思【核心梗】？"（引导去 `ideation` 步骤 1/2）

### 场景 2: "我有一个绝妙的核心梗/金手指" (Idea First)
*   **动作**：
    1.  直接调用 `novel-ideation` 的 **步骤 3 (金手指设计)** 或 **步骤 1 (创意脑暴)**。
    2.  生成对应设定文件。
    3.  **引导回流**："核心卖点已确立。接下来我们要围绕这个卖点搭建【大纲】还是先丰富一下【配角】？"

### 场景 3: "我想直接写个开头试试" (Draft First)
*   **动作**：
    1.  **警告**："直接开写容易导致后期崩盘，但为了捕捉灵感，我们可以先写【试读样章】。"
    2.  创建 `4-正文/试读样章.md`。
    3.  调用 `novel-write` 的辅助写作功能。
    4.  **引导回流**："样章写得不错。为了把这个故事写长，建议我们现在回头补全【大纲】和【人设】，把这个样章作为基准？"（引导去 `outline` 或 `define`）

---

## ✅ 结束检查
无论哪种模式，结束时必须确保：
1.  **SOLOENT.md** 已更新，反映了当前项目拥有的资产（设定/大纲/正文）。
2.  用户清楚**下一步**该去哪个标准模块（Analyze/Ideation/Outline/Write）。
