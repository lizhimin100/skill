# Workflow: Review (全流程动态质检)

> **⚠️ 说明**：本质检流程由 AI 生成，目前正根据武行个人创作经验进行深度优化。以下标准仅作为参考建议，请根据实际情况灵活调整。
> **目标**：在创作的每个关键节点（设定、大纲、正文）进行质量把控，防止问题累积。
> **原则**：早发现，早治疗。设定服务于剧情，剧情服务于爽点。

---

## 🛠️ 模块 1: 设定质检 (Setting Review)

**触发时机**：完成 `ideation` 流程，生成设定文件后。
**检查对象**：
- `2-设定/2.1_创意脑暴.md`
- `2-设定/2.2_新书设定案.md`
- `2-设定/2.3_金手指设定.md`
- `2-设定/2.4_主要角色设定表.md`

**质检标准 (Checklist)**：

1.  **逻辑自洽性 (Internal Logic)**
    - [ ] **力量体系闭环**：金手指的代价与收益是否平衡？后期是否会战力崩坏？
    - [ ] **世界观支撑**：地图广度与等级上限是否匹配？换地图逻辑是否顺畅？

2.  **爽点支撑度 (Pleasure Points)**
    - [ ] **金手指独特性**：是否只有主角能用？是否具备“人无我有，人有我优”的绝对优势？
    - [ ] **期待感设计**：金手指的进化路线是否清晰？是否有足够的升级期待？

3.  **利益驱动逻辑 (Motivation)**
    - [ ] **主角驱动力**：主角的目标是否足够强烈（生存/复仇/长生）？
    - [ ] **反派智商**：反派的行为是否符合其利益最大化？拒绝无脑送经验。

4.  **人物立体度 (Character Depth)**
    - [ ] **反差萌/记忆点**：主角和核心配角是否有独特的标签或反差（A面/B面）？

**执行动作**：
- 若发现问题，直接在对话中指出，并协助用户修改对应设定文件。
- 修改后，须再次确认是否引入新矛盾。

## 📋 模块 2: 大纲质检 (Outline Review)

**触发时机**：完成 `outline` 流程，生成总纲、卷纲或章纲后。
**检查对象**：
- `3-大纲/3.1_全书结构总纲.md`
- `3-大纲/XX卷_完整卷纲.md`
- `3-大纲/XX卷_章纲.md`

**质检标准 (Checklist)**：

1.  **节奏密度 (Pacing)**
    - [ ] **黄金节奏**：是否符合“三章一小高潮，十章一大高潮”的网文节奏？
    - [ ] **垃圾时间**：是否存在连续 2 章以上的铺垫或说明性文字？（须压缩或删除）

2.  **期待感管理 (Hooks)**
    - [ ] **钩子有效性**：每一章（特别是卷末/章末）是否留下了强悬念？
    - [ ] **伏笔埋设**：高潮前的铺垫是否足够？是否有生硬的机械降神？

3.  **爽点逻辑 (Payoff)**
    - [ ] **情绪释放**：压抑后的爆发是否足够酣畅？
    - [ ] **主角作用**：核心转折点是否由主角主导？（拒绝主角划水）

4.  **一致性检查 (Consistency)**
    - [ ] **设定对齐**：大纲剧情是否与 `2.2_新书设定案.md` 中的主线推演一致？
    - [ ] **战力检查**：本卷BOSS战力是否在主角当前等级的可应对范围内？

**执行动作**：
- 模拟【主编】视角，对大纲进行辣评。
- 提出具体的修改方案（如：调整事件顺序、增加危机感、强化金手指作用）。

## ✍️ 模块 3: 正文质检 (Draft Review)

**触发时机**：完成 `write` 流程，生成单章正文草稿后。
**检查对象**：`4-正文/第X章_草稿.md`

**质检标准 (Checklist)**：

1.  **黄金三章法则 (Opening)** (仅限开篇)
    - [ ] **切入点**：是否开篇即危机/冲突？（拒绝慢热）
    - [ ] **金手指**：金手指是否在合适时机（通常前3章）出现并发挥作用？
    - [ ] **代入感**：主角的处境是否能引起读者共情？

2.  **沉浸感 (Immersion)**
    - [ ] **Show, Don't Tell**：是否用动作、对话、细节描写代替了枯燥的设定说明？
    - [ ] **五感描写**：是否调动了视觉、听觉、嗅觉等感官体验？

3.  **情绪调动 (Emotion)**
    - [ ] **爽点/憋屈点**：读完本章，读者是感到爽、期待，还是憋屈？（憋屈不能过夜）
    - [ ] **装逼打脸**：如果有装逼情节，是否铺垫到位？打脸是否响亮？

4.  **文风与红线 (Style & Constraints)**
    - [ ] **AI味去除**：检查是否存在翻译腔、重复词汇、过度书面语。
    - [ ] **红线检查**：是否违反 `docs/MASTER.md` 中的禁忌？

**执行动作**：
- 模拟【毒舌读者】或【资深编辑】视角，进行段落级批注。
- 重点标注“出戏”、“啰嗦”、“逻辑不通”的地方，并提供改写示范。

## 🔄 模块 4: 设定回溯 (Feedback Loop)

**定位**：防止"剧情设定两张皮"，确保设定随剧情动态演进。

**触发时机**：
- 正文/大纲写作中涌现新灵感，与旧设定冲突时。
- 发现旧设定无法支撑当前剧情爽点时。

**执行动作**：
1.  **更新 Memory Bank**：
    - 补充新配角 -> `SOLOENT.md §3` & `§7.2`
    - 修正地图 -> `SOLOENT.md §2`
    - 更新伏笔 -> `SOLOENT.md §7.4`
2.  **更新设定文档**：
    - `2-设定/2.2_新书设定案.md` (主线/地图)
    - `2-设定/2.3_金手指设定.md` (补丁/进化)
    - `2-设定/2.4_主要角色设定表.md` (性格成长/关系变化)

- [ ] **正文层**：代入感强，情绪到位，文风达标。
- [ ] **数据层**：字数达标，Memory Bank 已更新。
