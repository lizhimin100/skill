# 章纲精修偏好清单（用户可配置模板）

> **定位**：这是你的个人创作偏好配置文件。在小说立项时填写一次，后续在每次章纲精修时由 AI 自动读取作为依据。
> **元化原则**：不绑定任何特定小说体系，所有字段均为"通用参数"而非"具体案例"。

---

## §7 章纲弹性度

> **定位**：定义你这本书的章纲"松紧度"。换卷、换书都可以重新设置。

### 7.1 精度模式

选择当前卷适用的章纲精度（默认：轻量级）：

- [ ] **骨架级**：每章只有情绪目标 + 核心事件（≤2个），不写剧情点。适用于"灵感驱动型"写作者。
- [x] **轻量级**（默认）：情绪目标 + 核心剧情点（1-3个）+ 节奏标签 + 留白提示。适用于大多数场景。
- [ ] **详细级**：情绪目标 + 核心剧情点 + 写法指导。仅适用于试错迭代型章节（习武/练招/实验过程），这些章节的高频操作需要细化表达。

### 7.2 切换规则

| 当前模式 | 何时可以切 | 切换到 |
|----------|-----------|--------|
| 骨架级 → 轻量级 | 连续多章发现灵感不够用，需要更多锚点 | 轻量级 |
| 轻量级 → 骨架级 | 感觉章纲还是太细，想进一步减负 | 骨架级 |
| 轻量级 → 详细级 | 某一章涉及高频操作（试错迭代），需要细化 | 详细级（仅该章） |
| 详细级 → 轻量级 | 高频操作章节完成 | 轻量级（恢复默认） |

> **说明**：精度模式可以在写正文过程中灵活切换，不需要重新立项。换卷时也可以调整。

## §1 流程类型（多选，至少选一项）

选择你这部小说的主体剧情推进模式：

- [ ] **试错迭代型**：主角通过反复实验/练习/调试来掌握技能或破解难题
  - 激活规则：必须包含"尝试→失败→调整→再尝试→成功"的完整迭代链
- [ ] **策略规划型**：主角通过制定方案→调配资源→执行来解决问题
  - 激活规则：必须包含"方案设计→资源评估→执行→复盘"的完整逻辑链
- [ ] **社交博弈型**：主角通过话术、情报、利益交换来推动剧情
  - 激活规则：必须包含"利益分析→话术设计→信息差利用→关系变化"
- [ ] **战斗执行型**：主角通过战斗/对抗来推动剧情
  - 激活规则：必须包含"冲突升级→对抗过程→胜负判定→后果影响"
- [ ] **探索发现型**：主角通过探索/解谜/发现来推动剧情
  - 激活规则：必须包含"线索发现→推理分析→风险尝试→收获/陷阱"

> **说明**：可以多选。例如"试错迭代型 + 社交博弈型"表示主角既要练功又要搞人际关系。

## §2 叙事风格参数

| 参数项 | 可选值 | 说明 |
|--------|--------|------|
| 叙事视角 | `限知主角（默认）` / `多POV切换` / `上帝视角` | 每章/每卷可以切换 |
| 对话占比 | `≥ X%`（建议 `≥ 40%`，可调） | 对话不是水字数，是制造节奏和节奏的工具 |
| 叙事手段比例 | `对话:X% / 描写:X% / 战斗:X% / 文戏:X%` | 确保不偏科 |
| 单章目标字数 | `2500` (默认，可调) | 用于容量预评估 |
| 章节标题风格 | `沿用章纲标题` / `口语化风格` / `无标题仅编号` | 与大纲文件的标题格式协调 |

## §3 创作分析偏好

### 3.1 动机关联
- [x] **主角内在动机**：主角做每一件事必须有内在驱动（忌惮/利益/风险/情感），不能只是"他做了"
- [ ] **反派动机**：反派的决策符合其利益最大化（可选启用）

### 3.2 过程细化
- [x] **操作细节**：写清楚具体怎么做的，不笼统写"做了某事"
- [x] **能力边界量化**：用量/范围/条件约束主角能做到什么
- [ ] **试错迭代**：如果涉及练手/实验/改造，必须有迭代过程
- [ ] **环境/规则制约**：写清楚客观条件对主角的限制

### 3.3 信息差处理
- [x] **信息差公式化**：按 `角色A + 不知道事实B + 导致了错误判断C` 标注
  - 若无 C（错误判断），不强写信息差
- [ ] **多势力视角检查**：涉及两个及以上势力时，主动提问是否切换视角

### 3.4 收益与结果
- [x] **收益类型**：收益不限于等级/经验，可以是熟练度/流程优化/知识/展示效果/人脉/声望/情报
- [x] **收益可视化**：写清楚"有什么用"，不抽象承诺

### 3.5 配角处理
- [x] **位置变化**：写清楚配角从哪儿到哪儿
- [x] **反应具体化**：配角对主角行为的反应要具体（行为/对话/表情），不笼统写"有反应"

### 3.6 容量与节奏
- [x] **容量预评估**：逐章评估梗概的事件密度与细节量是否足以支撑目标字数
- [ ] **节奏密度检查**：是否存在连续 2 章以上的铺垫或说明性文字（可选启用）

## §4 禁止行为（精修时严禁触发的操作）

- ❌ 跳过提问直接修改
- ❌ 将用户未提供的信息自行脑补后写入
- ❌ 对上下文档案中已明确的数据（如人数、等级）再次提问
- ❌ 在发现容量不足时，不提问而自行凭空捏造情节来凑字数
- ❌ 在提问的同时输出任何修改内容（必须先交互，再输出最终版）

---

## §5 精修输出格式

**精修完成后**，按以下格式输出：

```
## 第X章：[标题]
[精修后的章节梗概]
```

- 仅输出精修后的章节梗概
- 每个章节用 `## 第X章` 分隔
- 不附加修改说明、不附加分析、不附加评价
- 如果某章无需修改，直接输出原梗概（不加注释）

---

## §6 使用说明

1. **首次使用**：将此文件放入 `1-边界/` 目录，打开所有选项，根据本书类型勾选适用的流程类型和偏好项。
2. **增量修改**：写到中途发现偏好需要调整时（比如某本书后期从"试错迭代型"变成了"策略规划型"），直接修改此文件即可，不需要重写 skill 流程。
3. **换书时**：复制此文件到新小说的 `1-边界/` 目录，重置 §1 重新勾选即可。
4. **AI 读取时机**：在 outline.md 步骤3（章纲细化）精修子模块、以及 write.md 写作前置检查时，AI 必须读取此文件。
